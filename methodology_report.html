Methodology Report: E-Commerce Conversion Prediction

1. Goal

The objective of this project was to accurately predict whether a user will convert (binary classification) based on their browsing behavior and demographics. The primary metric optimized was the F1 Score.

2. Data Preparation

  • Combined Datasets: We merged the training data and public test data to maximize the amount of information the models could learn from.
  • Handling Missing Data: Missing numerical values were filled in using the median value of their respective columns.
  • Encoding: Text-based categories (like Device Type and Traffic Source) were converted into numbers so the models could understand them.

3. Feature Engineering

We created a few simple but powerful features to capture how users interact with the website:

  • Engagement Score: Multiplied pages viewed by products viewed to measure total user interest.
  • Time Per Page: Calculated the average time spent on each page to understand user engagement efficiency.

4. Machine Learning Models

We used an ensemble approach, combining two powerful algorithms to make the final decision:

  • XGBoost (Weighted 66%): A highly tuned gradient boosting model. It was given more weight in the final vote because it proved to be slightly stronger in our testing.
  • Random Forest (Weighted 33%): A balanced random forest model designed to handle class imbalances and provide stability to the predictions.

5. Optimization and Final Prediction

  • Cross-Validation: We tested our models rigorously using 5-fold cross-validation to ensure they would perform well on unseen data.
  • Threshold Tuning: Instead of using the default 50% probability to decide if someone converts, we tested multiple probability thresholds. We found and applied the exact threshold that maximized our target F1 score.
  • Final Output: The two models generated a combined probability score. If this score crossed our optimized threshold, the final prediction was set to "Converted" (1), otherwise "Not Converted" (0).
